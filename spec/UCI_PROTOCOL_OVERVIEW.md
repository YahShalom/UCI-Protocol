# Universal Capability Interface (UCI)

## Overview

The Universal Capability Interface (UCI) is an open protocol designed to provide governance, policy enforcement, and verifiable execution evidence for AI agent systems.

UCI is not an agent runtime, nor is it a tool connectivity protocol. Instead, it is a specification for a governance layer that sits above these components, providing a standardized way to control and audit agent actions. Its purpose is to ensure that as agents become more autonomous, their actions remain secure, compliant, and trustworthy.

## The Problem

The AI agent ecosystem is developing at an accelerated pace. Agents can now:

*   Call tools and interact with external APIs.
*   Execute complex, multi-step workflows.
*   Communicate and delegate tasks to other agents.

However, this rapid growth has created a governance gap. There is currently no standard, interoperable way to:

*   **Verify** what actions an agent actually took to achieve an outcome.
*   **Enforce** security and operational policies across different agent systems.
*   **Produce** a tamper-evident, auditable record of an agent's execution history.

This absence of a standardized governance layer creates significant trust, security, and compliance challenges, hindering the adoption of agentic systems in mission-critical and enterprise environments.

## The UCI Solution

UCI introduces a structured and verifiable execution pipeline that wraps every agent action in a governance envelope.

The core pipeline is:
`IntentRequest` → `ExecutionPlan` → `Capability Execution` → `EvidenceChain` → `Outcome`

The purpose of each component is:

*   **`IntentRequest`**: A machine-readable object that captures the goal of an action, the actor initiating it, and any policy constraints that must be enforced.
*   **`ExecutionPlan`**: A sequence of concrete steps generated by a UCI Kernel, defining the exact capabilities needed to satisfy the `IntentRequest`.
*   **`Capability Execution`**: The invocation of the actual tools, APIs, or functions that perform the requested work.
*   **`EvidenceChain`**: A cryptographically signed, immutable log of all significant events and steps taken during the execution, from the initial intent to the final result.
*   **`Outcome`**: A final object that contains the result of the execution, including its status (success, failure, or denial) and any returned data.

## The Agent Stack

UCI is designed to complement, not replace, existing protocols and frameworks. It provides a governance layer that integrates with the broader agent ecosystem.

*   **Agent Runtimes** (e.g., LangGraph, CrewAI, AutoGPT) handle the internal logic of **reasoning and planning**.
*   **Tool Protocols** (e.g., MCP) handle the **connectivity** between agents and capabilities.
*   **Agent Messaging** (e.g., A2A) handles **communication** between independent agents.

**UCI** sits above these layers, providing **governance, verification, and execution evidence** across all of them.

## Control Plane Architecture

UCI functions as the **control plane** for agent execution, separating the *decision to act* from the *action itself*.

The control plane, managed by a UCI Kernel, handles:

*   **Policy Enforcement:** Validating that a requested action is permitted.
*   **Capability Verification:** Ensuring a capability is trusted before it is used.
*   **Evidence Generation:** Recording a verifiable log of the execution.
*   **Deterministic Replay:** Providing a mechanism to audit the evidence log.

The underlying execution systems (the tools, APIs, and databases) remain responsible for performing the actual work in the **data plane**.

## Evidence Chains

The `EvidenceChain` is the primary artifact produced by a UCI execution. It is a powerful data structure that provides:

*   **Auditable Execution History:** A complete, step-by-step record of an agent's actions.
*   **Tamper-Evident Logs:** Cryptographic signatures ensure that any modification to the evidence after the fact is detectable.
*   **Replay Verification:** The ability to programmatically "replay" the evidence to confirm that the recorded outcome is a legitimate result of the recorded steps.

## Deployment Models

UCI can be integrated into existing architectures using several deployment models:

*   **UCI Gateway:** A standalone service that acts as a governing proxy in front of an existing agent application.
*   **UCI Sidecar:** A companion service running alongside an agent in a containerized environment, monitoring its execution and producing evidence.
*   **Embedded UCI Runtime:** A library integrated directly into an agent framework, enabling native UCI compliance.

## Benefits

Adopting UCI provides significant benefits for the entire agent ecosystem:

*   **Trustworthy Agent Systems:** Creates agents that operate within verifiable and auditable boundaries.
*   **Auditable Automation:** Generates the immutable proof required for security reviews and regulatory compliance.
*   **Policy Enforcement:** Enables consistent application of security and operational policies across diverse agent frameworks.
*   **Enterprise Readiness:** Delivers the security, governance, and audit features necessary for enterprise deployment.
*   **Interoperability:** `EvidenceChain`s provide a standard "execution receipt" that can be understood across different systems.

## Long-Term Vision

The long-term vision is a mature and modular AI ecosystem where each layer of the stack has a clearly defined, standardized role.

*   **UCI:** The universal layer for **governance and execution evidence**.
*   **Agent Runtimes:** The engine for **reasoning and planning**.
*   **MCP:** The standard for **tool connectivity**.
*   **A2A:** The standard for **agent-to-agent communication**.

Together, these layers create a secure, interoperable, and scalable foundation for the future of autonomous systems.
